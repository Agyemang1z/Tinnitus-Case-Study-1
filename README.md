# Modeling Tinnitus Functional Index Reduction Using Supervised Machine Learning Algorithms

## Overview

This repository contains Python code and data for a tinnitus-related case study focused on modeling reduction in Tinnitus Functional Index outcomes using supervised machine learning. The repository supports reproducible analysis of predictive modeling in a clinical or case-study context.

## Repository

- **Repository name:** `Tinnitus-Case-Study-1`
- **Repository type:** Jupyter Notebook health machine learning repository
- **Repository link:** https://github.com/Agyemang1z/Tinnitus-Case-Study-1
- **Primary author:** Edmund Fosu Agyemang
- **Academic identifier:** ORCID: https://orcid.org/0000-0001-8124-4493

## Research Objectives

- Analyze tinnitus case study data using supervised learning algorithms.
- Model changes or reductions in Tinnitus Functional Index outcomes.
- Compare predictive performance across candidate algorithms.
- Provide transparent code and data resources for manuscript extension or replication.

## Analytical Workflow

1. Load `CaseStudy1New.csv` into the notebook.
2. Clean variables and define the outcome related to Tinnitus Functional Index reduction.
3. Fit selected supervised learning models.
4. Evaluate classification or regression performance depending on outcome specification.
5. Summarize variable importance or model interpretation outputs if implemented.
6. Prepare figures and tables for manuscript reporting.

## Reproducibility Guide

Run the project from a clean working directory. The following commands provide a suggested starting point:

```bash
git clone https://github.com/Agyemang1z/Tinnitus-Case-Study-1.git
cd Tinnitus-Case-Study-1
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install pandas numpy scikit-learn matplotlib jupyter
jupyter notebook "Case Studies 1 Codes.ipynb"
```

If file names include spaces, keep quotation marks around the file name when launching notebooks or scripts from the terminal.
